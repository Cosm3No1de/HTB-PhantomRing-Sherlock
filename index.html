T2

IP del C2

Objetivo: Identificar la dirección IP del servidor de comando y control al que el agente intenta conectarse.
Metodología: Extraemos todas las cadenas ASCII del binario con strings y filtramos aquellas que coinciden con el patrón de una IPv4 (cuatro números separados por puntos).
strings agent | grep -E '([0-9]{1,3}\.){3}[0-9]{1,3}'
192.168.56.1
Interpretación: La IP aparece como cadena legible, lo que indica que está hardcodeada en el binario. Es una IP privada, típica de entornos de pruebas o redes internas.
Mostrar IoC 🌐 192.168.56.1
T3

Puerto del C2

Objetivo: Determinar el puerto TCP utilizado para la comunicación con el C2.
Metodología: El puerto suele estar codificado en la estructura sockaddr_in y se convierte a orden de red mediante htons. Buscamos la llamada a htons@plt en el código desensamblado y observamos el valor inmediato que se pasa como argumento.
objdump -d agent | grep -A5 -B5 'htons@plt'
mov $0x115d,%edi
call htons@plt
Interpretación: 0x115d en hexadecimal equivale a 4445 en decimal. Ese es el puerto que se usa en la conexión.
Mostrar IoC 🔌 4445
T4

Tiempo de espera para reconexión

Objetivo: Saber cuántos segundos espera el agente antes de intentar reconectar tras un fallo.
Metodología: Buscamos la llamada a la función sleep() en el código ensamblador y extraemos su argumento (el número de segundos).
objdump -d agent | grep -A2 -B2 'call.*sleep' | grep -o '\$0x[0-9a-f]*'
$0x78
Interpretación: 0x78 = 120 decimal. Por lo tanto, el agente espera 120 segundos (2 minutos) antes de reintentar la conexión.
Mostrar IoC ⏱️ 120 segundos
