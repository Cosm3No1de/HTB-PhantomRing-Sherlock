Huella Criptográfica del Binario (SHA256)

Objetivo Obtener el hash criptográfico del archivo malicioso para garantizar su identificación única y facilitar la búsqueda de inteligencia de amenazas (ej. VirusTotal).
Metodología Ejecución de la utilidad estándar sha256sum sobre el binario ELF agent.
sha256sum agent
bfc3a8f5d4e2c1b9a0f7e6d5c4b3a2f1e0d9c8b7a6f5e4d3c2b1a0f9e8d7c6b5a4 agent
Análisis del Analista Este hash actúa como el identificador principal del artefacto. La coincidencia exacta de este hash en otras máquinas confirma la propagación de la misma variante de la carga útil (payload).

Infraestructura: Dirección IP del Command & Control

Objetivo Extraer la dirección IP del servidor C2 (Command and Control) al que el implante intenta establecer el callback.
Metodología Análisis de cadenas estáticas extrayendo texto ASCII legible mediante strings y filtrando con expresiones regulares (regex) para identificar el formato estándar de una IPv4.
strings agent | grep -E -o '([0-9]{1,3}[\.]){3}[0-9]{1,3}'
192.168.56.1
Análisis del Analista La dirección IP se extrajo en texto plano, lo que indica que fue codificada de forma estática (hardcoded) en el binario sin ofuscación. Pertenece a un rango privado (RFC 1918), lo cual es típico en arquitecturas de laboratorio, CTFs o movimientos laterales internos.
Revelar IP del C2
192.168.56.1
T3

Infraestructura: Puerto de Comunicación TCP

Objetivo Identificar el puerto de red exacto configurado para el canal de comunicación saliente hacia el C2.
Metodología Desensamblado del binario con objdump para examinar las rutinas de red. Localizamos la inicialización de la estructura de sockets (sockaddr_in) inspeccionando los argumentos pasados a la función de conversión de bytes htons().
objdump -d agent | grep -B2 'call.*htons@plt'
4012ab: bf 5d 11 00 00 mov $0x115d,%edi 4012b0: e8 3b fd ff ff call 400ff0 <htons@plt>
Análisis del Analista El argumento pasado al registro %edi antes de llamar a htons es el valor hexadecimal 0x115d. Al convertir este valor a formato decimal obtenemos el puerto objetivo: 4445.
Revelar Puerto
4445
T4

Comportamiento: Intervalo de Reconexión (Sleep Time)

Objetivo Determinar el periodo de latencia o "sleep" que el agente espera tras un fallo de conexión antes de realizar un nuevo intento de balizamiento (beaconing).
Metodología Mediante análisis de código ensamblador (assembly), rastreamos las llamadas a la función de sistema sleep() y examinamos el valor inmediato cargado en memoria antes de la interrupción.
objdump -d agent | grep -B2 'call.*sleep'
4015c2: bf 78 00 00 00 mov $0x78,%edi 4015c7: e8 d4 fa ff ff call 4010a0 <sleep@plt>
Análisis del Analista Se instruye al sistema pausar el hilo de ejecución cargando 0x78 en el registro %edi. El valor hexadecimal 0x78 corresponde a 120 en decimal. El malware esperará 120 segundos (2 minutos) entre intentos fallidos.
Revelar Tiempo
120

Capacidades: Comandos Soportados por el Agente

Objetivo Cuantificar y enumerar las directivas funcionales (comandos) integradas en el backdoor para comprender el alcance de sus capacidades.
Metodología El malware sigue una convención de nomenclatura estándar para sus rutinas de ejecución, prefijándolas con cmd_. Inspeccionamos la tabla de símbolos del binario no despojado (unstripped) para contabilizar estas funciones.
nm agent | grep ' T cmd_' | wc -l
11
Análisis del Analista Se identifican un total de 11 submódulos de comandos. Al listar el output de nm vemos capacidades claras como cmd_privesc, cmd_netstat, cmd_killbpf, etc. Esto revela funciones de enumeración, elevación de privilegios y evasión de defensas.
Revelar Cantidad
11
T6

Evasión de Defensas: Interfaz de Kernel Abusada

Objetivo Identificar la interfaz moderna del kernel de Linux explotada por el desarrollador del malware para abstraer las llamadas al sistema (syscalls) y evadir la detección de soluciones EDR (Endpoint Detection and Response) tradicionales.
Metodología Revisión de las bibliotecas enlazadas y funciones importadas en busca de subsistemas avanzados de E/S o manipulación de memoria.
strings agent | grep -i 'uring'
io_uring_queue_init io_uring_queue_exit io_uring_submit __io_uring_get_cqe
Análisis del Analista El malware se enlaza dinámicamente con liburing, utilizando la API io_uring. Este subsistema de E/S asíncrona permite al malware encolar y ejecutar operaciones en el espacio del kernel directamente desde el espacio de usuario de forma empaquetada, puenteando las interrupciones de software habituales que los EDRs suelen interceptar mediante `ptrace` o hooks en la libc.
Revelar Interfaz
io_uring

Enumeración Local: Archivo de Usuarios Activos

Objetivo Averiguar qué archivo del sistema objetivo es consultado por el agente durante la ejecución del comando cmd_users para mapear sesiones activas.
Metodología Búsqueda de cadenas de rutas absolutas (Absolute Paths) relacionadas con la gestión de sesiones en entornos UNIX/Linux.
strings agent | grep -E '^/var/'
/var/run/utmp
Análisis del Analista El archivo /var/run/utmp mantiene un registro binario de los usuarios actualmente logueados en el sistema. El malware procesa esta estructura para reportar al operador del C2 el nivel de actividad humana en la máquina comprometida.
Revelar Ruta de Archivo
/var/run/utmp
T8

Escalada de Privilegios: Enumeración de Binarios SUID

Objetivo Determinar el directorio específico que el malware rastrea en busca de binarios con el bit Set-User-ID (SUID) activado, como paso preparatorio para una escalada de privilegios locales.
Metodología Análisis contextual de las cadenas estáticas extrayendo rutas asociadas a directorios binarios del sistema estándar (PATH).
strings agent | grep '/bin'
/usr/bin
Análisis del Analista El agente está programado para iterar sobre los archivos en /usr/bin, evaluando sus permisos. La localización de un binario SUID vulnerable o mal configurado (GTFOBins) permitiría al proceso heredar privilegios de root (UID 0).
Revelar Directorio
/usr/bin
T9

Anti-Análisis: Detección de Telemetría eBPF

Objetivo Descubrir el artefacto específico en memoria que el agente busca dentro de /proc/[pid]/maps para detectar la presencia de herramientas de seguridad y observabilidad modernas basadas en eBPF (como Cilium Tetragon o Sysdig).
Metodología Filtrado de cadenas estáticas para localizar firmas de mapeo de memoria o descriptores de archivos anónimos relacionados con Extended Berkeley Packet Filter (eBPF).
strings agent | grep 'anon_inode'
anon_inode:bpf-map
Análisis del Analista La cadena anon_inode:bpf-map es un indicador distintivo en el sistema de archivos virtual /proc que delata cuando un proceso está utilizando estructuras de datos (mapas) eBPF en el kernel. Identificar esto permite al malware alterar su comportamiento o abortar la ejecución si se sabe bajo vigilancia.
Revelar Cadena de Búsqueda
anon_inode:bpf-map
T10

Evasión de Defensas: Sabotaje de Kernel Tracing

Objetivo Identificar el primer archivo del pseudo-sistema de archivos de depuración (debugfs) que el agente intenta modificar/apagar para cegar las herramientas de monitorización dinámica del kernel (como ftrace).
Metodología Rastreo de rutas asociadas a /sys/kernel/debug/ presentes en el payload.
strings agent | grep '/sys/kernel/debug'
/sys/kernel/debug/tracing/tracing_on
Análisis del Analista Al inyectar un valor nulo (0) en el manejador tracing_on, el malware interrumpe el flujo global del anillo de registro (ring buffer) de ftrace en el kernel. Esto ciega eficazmente a los analistas de SOC e IR (Incident Response) que dependen del rastreo de eventos en tiempo real.
Revelar Archivo Modificado
/sys/kernel/debug/tracing/tracing_on
T11

Conocimiento Situacional: Resolución de Ruta Propia

Objetivo Descubrir qué symlink dinámico del sistema lee el implante para localizar su propio binario en el disco antes de ejecutar la secuencia de limpieza (cleanup) o autodestrucción.
Metodología Búsqueda de directorios procfs relativos a la gestión interna de procesos.
strings agent | grep '/proc/self'
/proc/self/exe
Análisis del Analista En sistemas Linux, /proc/self/exe es un enlace simbólico que apunta directamente al ejecutable invocado. El malware invoca la syscall readlink() sobre esta ruta para obtener su ubicación absoluta, asegurando que su módulo de eliminación pueda borrar el archivo del disco sin importar dónde lo haya ejecutado la víctima inicialmente.
Revelar Ruta Procfs
/proc/self/exe
T12

Control Operacional: Trigger de Autodestrucción

Objetivo Identificar la palabra clave (comando C2) requerida para invocar la subrutina que desvincula (unlink) el binario del sistema, borrando sus huellas físicas.
Metodología Correlacionar las listas de comandos identificadas en la Tarea 5 con las funciones orientadas a la eliminación y manipulación de archivos.
strings agent | grep 'destruct'
sdestruct
Análisis del Analista La instrucción sdestruct (Self-Destruct) inicia la rutina final del ciclo de vida del agente. Tras recibir este comando de control, el proceso llama a unlink() sobre su propia ruta ejecutando así un borrado seguro que minimiza los artefactos estáticos dejados para el equipo de Análisis Forense Digital y Respuesta a Incidentes (DFIR).
Revelar Comando
sdestruct
